# 🧾 Projet SQL – Analyse des Ventes (Base Chinook)

## 🎯 Objectif
Mettre en pratique les fondamentaux du langage SQL à travers l’analyse des ventes d’une base musicale fictive (*Chinook*). Ce projet démontre ma capacité à :
- Écrire des requêtes SQL efficaces (jointures, agrégats, conditions…)
- Extraire des KPI pertinents à partir d’une base relationnelle
- Organiser les résultats pour une analyse métier (ventes, clients, agents, produits)

---

## 🧠 Données utilisées
**Base de données Chinook** (SQLite), comprenant :
- Clients (`Customer`)
- Factures (`Invoice`)
- Détails de factures (`InvoiceLine`)
- Employés / agents de vente (`Employee`)

---

## 🔍 Requêtes réalisées

| Requête | Description |
|--------|-------------|
| 1 | Liste des clients non américains |
| 2 | Liste des clients brésiliens |
| 3 | Factures des clients brésiliens |
| 4 | Liste des agents de vente |
| 5 | Pays uniques dans les factures |
| 6 | Factures par agent de vente |
| 7 | Détail des factures avec clients et agents |
| 8 | Nombre de factures et total des ventes par année (2009 & 2011) |

---

## 🔧 Technologies utilisées
- SQL (SQLite)
- DB Browser for SQLite
- TextEdit (édition de requêtes et README)

---

## ✨ Résultat
Un jeu de requêtes commentées, reflétant une maîtrise des bases SQL pour la data analysis :
- Suivi du chiffre d’affaires
- Analyse par pays, client, agent
- Lecture détaillée des factures
- Requêtes prêtes pour reporting ou BI
